---

OpenBeavs is available as a live web application for the Oregon State University community. You only need a modern browser and your OSU Microsoft account to get started. This guide walks you through signing in, finding an agent, and having your first conversation — everything runs in the cloud, so there is nothing to install on your machine.

<Note>
  The live deployment requires an OSU Microsoft account. If you want to run OpenBeavs without OSU authentication, see the [local setup guide](/local-setup) instead.
</Note>

## Access the live app

<Steps>
  <Step title="Open OpenBeavs in your browser">
    Navigate to the OpenBeavs live deployment. The app is hosted on Google Cloud Run and is accessible from any modern browser — Chrome, Firefox, and Safari are all supported.
  </Step>

  <Step title="Sign in with your OSU account">
    Click **Sign in with Microsoft** on the login screen. You will be redirected to the standard OSU Microsoft SSO flow. Enter your OSU email address (e.g. `yourname@oregonstate.edu`) and your OSU password. No new account is needed — your existing OSU credentials work directly.

    <Info>
      If your institution requires multi-factor authentication, complete the MFA prompt as usual before being redirected back to OpenBeavs.
    </Info>
  </Step>

  <Step title="Land on the main chat interface">
    After signing in you will arrive at the main chat interface. This is where all your conversations with agents happen. You can start a new chat at any time by clicking **New Chat** in the left sidebar.
  </Step>
</Steps>

## Browse the agent registry

<Steps>
  <Step title="Open the agent registry">
    Click the **Workspace** icon in the left navigation sidebar, then select **Agents**. This opens the agent registry — a grid of all agents that have been published to the OpenBeavs hub by OSU teams and developers.
  </Step>

  <Step title="Explore available agents">
    Each card in the registry shows the agent's name, description, and the skills it exposes. Current agents include:

    - **OSU Expert** — answers questions about OSU courses, departments, admissions, financial aid, and campus life using a live-crawled knowledge base.
    - **Cyrano de Bergerac** — a writing coach that helps craft eloquent responses.
    - **Weather Expert** — provides weather information.
    - **Master Router** — an orchestrator that routes queries to the most appropriate specialist agent automatically.

    <Tip>
      Look at the tags on each agent card to find agents relevant to your needs. Tags like `rag`, `osu`, and `writing` help you filter quickly.
    </Tip>
  </Step>
</Steps>

## Install your first agent

<Steps>
  <Step title="Select an agent to install">
    Click on any agent card in the registry to open its detail page. Review the agent's description and listed skills to confirm it is the right one for you.
  </Step>

  <Step title="Add the agent to your workspace">
    Click the **Install** button on the agent detail page. The agent is now active in your workspace and will appear in the model selector dropdown when you start a new chat.
  </Step>
</Steps>

## Send your first message

<Steps>
  <Step title="Start a new chat">
    Click **New Chat** in the left sidebar to open a fresh conversation.
  </Step>

  <Step title="Select your installed agent">
    At the top of the chat window, click the model dropdown. Your installed agent will appear in the list alongside any other models configured on the hub. Select it.

    <Info>
      Agents are tagged with `agent` and `a2a` labels in the model dropdown so they are easy to distinguish from standard LLM models.
    </Info>
  </Step>

  <Step title="Type and send your message">
    Type your question or request in the message box and press **Enter** or click the send button. The agent will process your message and respond. For the OSU Expert agent, try something like:

    > "What are the requirements for the CS degree at OSU?"

    or

    > "When is the deadline to apply for financial aid this year?"
  </Step>

  <Step title="Continue the conversation">
    You can continue chatting in the same thread. Agents maintain conversation context, so follow-up questions work naturally. If you want a fresh start, open a new chat at any time.
  </Step>
</Steps>

## Connect an A2A agent (admin only)

If you are an admin and want to connect an external A2A-compatible agent to the hub — for example, an agent you have deployed yourself — use the Admin Settings panel.

<Steps>
  <Step title="Open Admin Settings">
    Click your profile icon in the top-right corner, select **Settings**, then navigate to **Admin Settings** > **Connections**.
  </Step>

  <Step title="Enable A2A agents">
    Scroll down to the **A2A Agents** section. Toggle the switch to **ON** to enable A2A agent connections.
  </Step>

  <Step title="Add a new agent connection">
    Click the **+** button to add a new connection. Enter the agent's URL, for example:

    ```
    localhost:8001
    ```

    Leave the name field blank to auto-detect the agent's name from its `/.well-known/agent.json` discovery card, or enter a custom display name.
  </Step>

  <Step title="Verify and save">
    Click the **Verify Connection** button (the refresh icon). Wait for the green checkmark to confirm the hub can reach the agent and read its discovery card. Then click **Save** at the bottom of the page.

    <Warning>
      If you see a red X instead of a green checkmark, confirm the agent server is running and reachable. For localhost agents, try both `localhost:8001` and `http://localhost:8001`.
    </Warning>

    Once saved, the agent will appear in the model selector in any new chat.
  </Step>
</Steps>
